# avatarsquare

avatarsquareは、アバターを使い、リアルタイムに他の人と交流できるwebアプリケーションです。
league of legendsのような見下ろし型の視点で、アバターを操作しながら、他のユーザーとチャットやゲームを楽しむことができます。

通信にはwebRTCを使用します。ユーザーは自分のアバターの描画結果と、現在位置などをストリームとして送信し、他のユーザーはそのストリームを受信して表示します。これにより、リアルタイムでの交流が可能になります。これによりアバターの3Dモデルを送信することなく他人に表示できるため、安全にアバターを共有することができます。

## だいじなこと
- アバターの3Dモデルを送信することなく、描画結果をストリームとして送信することで、他人に安全にアバターを共有できるようにします。
- 拡張性が大事です。例えばすべての操作をコマンドというインターフェースに統一することで、新しく機能を追加した際も、それにアクセスしやすくなります。
    - 厳密さよりも拡張性のほうが大事です。avatarsquareは厳密なゲームというよりはコミュニケーションツールであるため、ユーザーのチートを防止することよりも、ユーザーが自由に楽しめることを優先します。
- 本当はユーザーが自由にプラグインやマップサーバーを開発して、アバターのアクションやゲームのルール・マップなどを追加できるようにしたいです。ユーザーが自由にプラグインを開発できるようにすることで、avatarsquareはより多様な楽しみ方ができるようになります。

## 開発方針

###  Phase 1. アバターの描画と移動
まずは、アバターの描画と移動を実装します。アバターは、VRM形式のものを読み込めるようにします。
アバターの移動には右クリックでの移動を実装します。ユーザーが右クリックした位置にアバターが移動するようにします。
簡易的なマップを用意し、アバターがそのマップ上で移動できるようにします。
マップは、ある程度前後感のあるマップだと尚良いです。

### Phase 2. アバターのストリーム送信と受信
次に、アバターの描画結果をストリームとして送信し、他のユーザーがそのストリームを受信して表示できるようにします。
ユーザーがアバターを操作すると、その描画結果がリアルタイムで他のユーザーに送信され、他のユーザーはその描画結果を受信して表示します。これにより、他のユーザーとリアルタイムで交流できるようになります。

### Phase 3. アバターアクションの実装
アバターのアクションを実装します。例えば、ジャンプや攻撃などのアクションを実装し、ユーザーがアバターを操作することで、アクションを実行できるようにします。アクションの実行結果もリアルタイムで他のユーザーに送信され、他のユーザーはそのアクションの結果を受信して表示します。これにより、アバターのアクションを通じて、他のユーザーとよりインタラクティブに交流できるようになります。

### Phase 4. コマンド機能の実装
操作をすべてコマンド化します。たとえば、移動や攻撃などのアクションの実行をコマンドとして定義し、ユーザーがマクロ化できるようにします。これにより、ユーザーは複雑な操作を簡単に実行できるようになります。

### Phase 5. HUDの実装
HUD（Head-Up Display）を実装します。リファレンスとしては、FF14のHUDを参考にします。HUDには、アバターのステータスや現在位置、チャットウィンドウなどを表示します。これにより、ユーザーはアバターの状態や他のユーザーとの交流状況をリアルタイムで把握できるようになります。
ホットバーを実装し、アクションボタンを画面上に配置します。また、ホットバーにはキーボードのボタンを割り当てできるようにします。

### 開発環境の起動

```sh
docker compose up -d      # LiveKit (SFU) + トークン発行API
cd client && pnpm dev     # クライアント (Vite)
```

トークンAPIだけ手元で動かしたい場合は `go run ./server` でも起動できます(ポート8787)。

ブラウザで開くと自動でルーム`square`に入室します(`?room=xxx`で変更可)。

トークンサーバーのエンドポイントは既定で`https://avatar-square.tunnel.anthrotech.dev/token`に接続します。
画面上の入力欄(localStorageに保存)、`?endpoint=`クエリ、環境変数`VITE_TOKEN_URL`のいずれでも変更できます
(優先順: クエリ > 入力欄 > 環境変数 > 既定値)。ローカル開発時は`http://localhost:8787/token`を指定してください。
`.vrm`をドラッグ&ドロップでアバター読み込み、`.vrma`/Mixamo系`.fbx`でモーション登録ができます。

## Phase 6. ゲーム機能の実装
ゲーム機能を実装します。ここでは、lolのドレイクみたいなボスを実装し、ユーザーが協力してボスを倒すことができるようにします。ボスの攻撃や行動パターンを実装し、ユーザーが戦略を立てて協力して戦うことができるようにします。ボスのHPや状態などもHUDに表示されるようにします。
