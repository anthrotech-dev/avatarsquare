# avatarsquare server update

新しいアップデートとして、ワールドという概念を追加する。
ワールドはいわゆる「ルーム」や「セッション」や「インスタンス」のようなもの。

現在おそらくクライアント側で持っているマップの情報を、サーバーから提供するようにする。
サーバーは、そのサーバーが提供しているワールドの一覧を返す。そのワールドに接続すると、クライアントはそのワールドのマップ情報を取得し、ユーザーはそのワールド内でアバターを操作できるようになる。ワールド切り替え時には画面はロードに入る。

ワールドのデータ構造は、基本的にはjsonで表現される。
また、ワールド内のギミック当(例えば、ドアやスイッチ・NPCの体力など)の状態もサーバー側で管理される。クライアントはワールドに接続した際に、ワールドの状態を取得し、ユーザーが操作するアバターやギミックの状態をサーバーに送信することで、他のユーザーと同期される。
ギミックは、wasmで記述されたものをワールドデータとしてサーバーに読み込ませる。(ワールドデータのjsonに、ギミックのwasmファイルへのURLを記述する)

サーバー管理者は、信頼できるワールドのURLのみを選択し、サーバーの起動設定にて、ワールドのURLを指定することで、そのワールドを提供することができるようにする。

## 作成するサンプル
まず現在のクライアントにハードコードされたワールドの情報を、サーバーから取得するように変更する。
ワールド情報は一旦このwebサイトのpublicに置いておくことでそのまま提供できるはず。

さらに、このワールドにギミックを追加する。たとえば、いま攻撃モーションだけあるが、攻撃を実際に受けるオブジェクトがない。かかしを新しくワールドに作成し、攻撃を受けるとHPが減るようにする。

また、クリックでインタラクトできるオブジェクトも追加する。例えばボタンを配置し、置くとカウンターが増加するようにする。
